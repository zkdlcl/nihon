# 🚀 AI-Native 언어 학습 프레임워크 제안서

현재 우리의 프로젝트 구조도 훌륭하지만, 최근 AI 커뮤니티와 '하네스 엔지니어링(Harness Engineering - AI 에이전트를 효과적으로 묶고 활용하는 기술)' 트렌드를 반영하여, 우리의 일본어 학습 환경을 **'AI-Native 학습 에이전트 시스템'**으로 대폭 업그레이드할 수 있는 방안을 제안합니다.

이 제안은 사용자가 단순히 AI에게 물어보는 수동적 학습이 아니라, AI 에이전트들이 주도적으로 학습 데이터를 생성, 관리, 시험, 피드백하는 능동적이고 상호작용적인 시스템(Multi-Agent System)을 구축하는 데 목적이 있습니다.

---

## 💡 1. Multi-Agent 분업화 도입 (Harnessing Specialized Agents)

현재는 하나의 AI(Antigravity 또는 Claude/Gemini)가 일인 다역을 하고 있습니다. 하네스 엔지니어링의 핵심은 프롬프트 체인이나 다중 에이전트 워크플로우(예: Microsoft AutoGen, CrewAI 개념)를 도입하여 AI의 역할을 세분화하는 것입니다.

### 제안하는 에이전트 페르소나 (Skill/Prompt 분리)
우리 시스템의 `.agent/skills/` 디렉토리에 다음 스킬들을 추가하여 역할을 명확히 분리합니다.

1.  👩‍🏫 **문법/회화 튜터 에이전트 (`grammar-tutor`)**
    *   **역할**: 개념 설명, 뉘앙스 차이 설명, 장문 교정 및 회화 연습 파트너. 가장 친절하고 상세한 피드백 제공.
2.  📝 **퀴즈 마스터 에이전트 (`quiz-master`)**
    *   **역할**: 학습 로그(Daily-Logs)와 오답 노트 분석 후 맞춤형 문제 은행 생성. 객관식, 빈칸 채우기, 청해/독해 지문 자동 생성.
3.  🗂️ **지식 베이스 관리자 에이전트 (`kb-manager`)**
    *   **역할**: Anki 호환 형식(CSV 등)으로 단어장 자동 변환, 중복 단어 체크, 누적 단어 기반으로 복습 주기(Spaced Repetition) 계산 및 알림 파일(예: `今日_의_복습.md`) 생성.

## 🛠️ 2. 구조적 데이터 파이프라인 (Data-Driven Learning)

학습 로그를 단순히 마크다운 텍스트로만 두지 않고, AI가 더 잘 파싱하고 분석할 수 있도록 메타데이터(Frontmatter)와 체계적인 데이터 구조를 도입합니다.

### A. YAML Frontmatter 도입
새로운 `Daily-Logs` 파일들은 Obsidian의 Dataview 플러그인이나 AI가 쉽게 읽을 수 있도록 상단에 YAML 메타데이터를 추가합니다.

```yaml
---
date: 2026-04-21
day_number: 14
tags: [일본어, N5, 의문사, 복습]
mastery_score: 85
difficulties: [과거부정형, 郵便局 한자]
status: completed
---
```
*기대 효과*: AI가 'difficulties' 배열만 긁어와서 다음 날 아침 맞춤형 복습 문제를 던져줄 수 있습니다.

### B. Anki/Quizlet 연동 포맷 자동화
AI가 단어를 정리할 때 Markdown 표뿐만 아니라, `import_anki.csv` 같은 플레인 텍스트 파일을 자동으로 함께 생성하도록 스킬을 업데이트합니다. 이를 통해 손쉽게 Anki나 암기 앱으로 데이터를 넘길 수 있습니다.

## 🧠 3. RAG (Retrieval-Augmented Generation) 최적화

하네스 엔지니어링의 또 다른 축은 RAG입니다. AI가 사용자의 과거 수십 일치 로그를 일일이 읽으면 컨텍스트 창이 낭비되거나 핵심을 놓치기 쉽습니다.

*   **해결책**: 매주 주말(예: Day 7, Day 14)마다 기존 `Daily-Logs`를 분석해 핵심만 압축한 **`User-Brain-Profile.md`**를 자동 생성/업데이트하게 합니다.
*   **프로필 구성 내용**:
    *   마스터한 문법 포인트 (예: ~が好き, ~は~です)
    *   반복적으로 틀리는 아킬레스건 (예: い형용사/な형용사 과거형 변환)
    *   사용자의 관심사 기반 어휘 (최근 애니메이션 용어, IT 용어 등)
*   **활용**: AI 어시스턴트는 매 수업 시작 시 방대한 전체 로그 대신 이 `User-Brain-Profile.md` 하나만 참조(RAG)하면, 곧바로 사용자의 현재 수준과 약점에 맞는 완전히 개인화된 수업을 진행할 수 있습니다.

## 🔍 4. 적극적(Proactive) 학습 트리거와 상황극 (Roleplay)

사용자가 "오늘 공부할래" 하고 시작하는 것을 넘어, 하네스 시스템이 먼저 상황을 제시합니다.

*   **상황극(RPG) Prompting**: 단순히 문장을 주고 번역하라고 하지 않습니다.
    *   *AI 제시*: "당신은 지금 도쿄 여행 중입니다! 신주쿠역에서 길을 잃었어요. 앞에 있는 친절해 보이는 경찰관(저)에게 어제 배운 '어디(どこ)'와 '가고 싶다(~たい)'를 써서 질문해 보세요!"
*   **멀티모달 대비 (Future-proofing)**: 향후 AI가 브라우저 서치나 이미지 분석을 지원하므로, 실제 일본 웹사이트 캡처 화면이나 메뉴판 사진(이미지 파일)을 `Resources` 폴더에 넣으면, AI가 해당 이미지를 분석하여 즉석에서 실전 메뉴 주문 연습을 시키는 워크플로우를 준비할 수 있습니다.
